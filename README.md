# 🛠 Повний технічний огляд та інструкція (3D Physics Sandbox)

Цей проєкт — це сучасний веб-додаток на базі **React**, **Three.js** (для 3D графіки) та **Rapier** (для фізики). Він використовує інструмент збірки **Vite**, який забезпечує миттєве оновлення в браузері під час розробки та швидку компіляцію для фінального релізу.

---

## 🏗 Архітектура та Логіка (Як це працює)

### 1. Система сітки (Grid) та Клітинки (Cells)
*   Світ будується на основі сітки. Кожна клітинка (`cell`) має розмір 3x3 метри (константа `TILE = 3`).
*   Клітинки визначаються в `layout.cells` (у файлі `src/store.ts`). Кожна клітинка має координати `x`, `y` та `type` (1=підлога, 2=двері, 3=замкнені двері).

### 2. Структури (Structures)
Існує два типи структур:
*   **Зовнішні стіни (Outer Walls):** Генеруються автоматично навколо периметра клітинок.
*   **Внутрішні структури (Internal Structures):** Створюються гравцем за допомогою режиму будівництва (`BuildSystem.tsx`). Зберігаються в `useStore.internalStructures`.

### 3. Логіка вирізання отворів (Вікна та Двері)
Щоб уникнути візуальних артефактів (Z-fighting) і дозволити світлу проходити крізь вікна, стіни не малюються як суцільні блоки, якщо в них є отвори.
*   **Файл:** `src/components/Room.tsx`
*   **Логіка:**
    1.  Для кожної стіни (зовнішньої чи внутрішньої) система шукає всі вікна та двері, які до неї прив'язані (`parentWallId`).
    2.  Обчислюються "інтервали" (відрізки), де знаходяться ці отвори.
    3.  Стіна розбивається на сегменти (`SolidWall`), які малюються *між* цими інтервалами.
    4.  Над отворами малюються "перемички" (headers), а під вікнами — "підвіконня" (sills).
    5.  **Константи:** Ширина вирізу для вікна — `0.85`, для дверей — `1.0` (`DOOR_W`). Висота вирізу для вікна — `1.25`.

### 4. Логіка розсувних дверей
Двері (`InteractiveDoor`) автоматично визначають, чи можуть вони бути розсувними.
*   **Файл:** `src/components/InteractiveDoor.tsx` та `Room.tsx`
*   **Логіка:** Система перевіряє наявність вільного місця на стіні (мінімум `0.9` одиниць) зліва або справа від дверей. Якщо місце є, двері стають розсувними. Якщо місця немає (наприклад, поруч вікно), двері стають звичайними (на петлях).

### 5. Система Будівництва (BuildSystem)
*   **Файл:** `src/components/BuildSystem.tsx`
*   **Логіка:** Використовує `Raycaster` для визначення точки наведення.
*   **Обмеження:**
    *   Вікна та двері можна ставити тільки на існуючі стіни.
    *   **Перевірка відстані:** Не можна ставити об'єкти занадто близько один до одного. Для вікон потрібна відстань `0.9`, для дверей — `1.85` (щоб було місце для розсування).

### 6. Освітлення та Сонце (SunSystem)
*   **Файл:** `src/components/SunSystem.tsx`
*   **Логіка:** Час у грі (`gameTime`) керує позицією сонця (`directionalLight`).
*   Світло потрапляє в кімнату **тільки** через вікна або двері. Навколишнє світло (`ambientLight`) зведено до мінімуму, щоб створити реалістичні тіні.

### 7. Фізика (Rapier)
*   Стіни, підлога та стеля використовують `RigidBody type="fixed"`.
*   Гравець використовує `CapsuleCollider`.
*   **Важливо:** Промінь будівництва ігнорує капсулу гравця, щоб не можна було будувати на самому собі.

## 📂 Детальний огляд структури файлів

### 1. Коренева папка (Конфігурація)
*   `package.json` — **Головний файл проєкту**. Тут перелічені всі бібліотеки (dependencies), які потрібні для роботи, та команди (scripts) для запуску.
*   `vite.config.ts` — Налаштування інструменту Vite. Він каже браузеру, як обробляти React-файли та TypeScript.
*   `tsconfig.json` — Налаштування мови TypeScript (типізація, перевірка помилок).
*   `README.md` — Ця інструкція.
*   `.gitignore` — Список файлів, які не треба завантажувати в репозиторій (наприклад, папка `node_modules`).

### 2. Папка `public/` (Статичні ресурси)
Все, що лежить тут, доступне в браузері за прямим посиланням.
*   Сюди ви маєте класти свої 3D моделі (папка `models/`).
*   Сюди кладуться текстури, звуки та іконки.

### 3. Папка `src/` (Вихідний код)
Тут знаходиться вся логіка гри.
*   `main.tsx` — Вхідна точка. Вона каже React: "Візьми компонент App і встав його в HTML-сторінку".
*   `index.css` — Глобальні стилі (Tailwind CSS). Тут налаштовуються кольори та шрифти.
*   `store.ts` — **Управління станом (Zustand)**. Це "база даних" вашої гри в реальному часі. Тут зберігається:
    *   Що зараз у руках у гравця.
    *   Список усіх предметів у кімнаті.
    *   Логіка підняття/кидання предметів.
*   `App.tsx` — **Головний каркас**. Тут налаштовується 3D-сцена: небо, світло, фізичний світ та підключаються всі інші компоненти.

#### 🧩 Компоненти (`src/components/`)
*   `Player.tsx` — Логіка гравця: як він ходить, як крутить головою, як "стріляє" променем, щоб знайти предмети для підняття.
*   `Room.tsx` — Геометрія кімнати (стіни, підлога, освітлення). Відповідає за вирізання отворів для вікон та дверей.
*   `BuildSystem.tsx` — Логіка режиму будівництва (розміщення стін, вікон, дверей).
*   `InteractiveDoor.tsx` — Логіка дверей (відкривання/закривання, розсувні/на петлях).
*   `SunSystem.tsx` — Система дня і ночі, освітлення.
*   `SceneItem.tsx` — Універсальний шаблон для будь-якого предмета. Він знає, як малювати коробку, а як — 3D модель, і як надати їм масу.
*   `UI.tsx` — Все, що ви бачите "поверх" гри: приціл, меню інвентарю, кнопки керування.

## ⚙️ Як з цим працювати (Розробка та Компіляція)

### 1. Підготовка (Один раз)
Після того, як ви розпакували архів, вам треба встановити всі бібліотеки. У терміналі напишіть:
```bash
npm install
```
Це створить папку `node_modules`, де будуть лежати тисячі файлів, необхідних для роботи Three.js та React.

### 2. Режим розробки (Live Preview)
Щоб бачити зміни в реальному часі:
```bash
npm run dev
```
Vite запустить локальний сервер. Тепер, як тільки ви зміните будь-який рядок коду (наприклад, колір стіни в `Room.tsx`), браузер оновиться миттєво без перезавантаження сторінки.

### 3. Компіляція для клієнта (Build)
Коли ви закінчили розробку і хочете викласти гру в інтернет (наприклад, на GitHub Pages, Netlify або свій хостинг):
```bash
npm run build
```
**Що станеться:** Vite візьме сотні ваших файлів, оптимізує їх, стисне і перетворить на декілька маленьких файлів у папці `dist/`.
*   Папка `dist/` — це і є ваша готова гра. 
*   Ви можете просто завантажити вміст цієї папки на будь-який статичний хостинг.

### 4. Перевірка помилок (Lint)
Перед компіляцією корисно перевірити, чи немає в коді помилок типізації:
```bash
npm run lint
```

## 💡 Поради щодо роботи в браузері

1.  **Консоль розробника**: Натисніть `F12` у браузері. Там ви побачите помилки, якщо щось піде не так (наприклад, якщо 3D модель не завантажилася).
2.  **Додавання нових моделей**:
    *   Покладіть файл `my_sofa.obj` у `public/models/`.
    *   Відкрийте `src/store.ts`.
    *   Додайте новий об'єкт у масив `inventory` з типом `obj` та вкажіть шлях `/models/my_sofa.obj`.
3.  **Зміна фізики**:
    *   Хочете, щоб гравітація була як на Місяці? Відкрийте `src/App.tsx` і змініть параметр `gravity={[0, -9.81, 0]}` на `[0, -1.6, 0]`.
    *   Хочете змінити масу предмета? Змініть параметр `mass` у `src/store.ts`.

Ця структура дозволяє вам легко масштабувати проєкт: додавати нові кімнати, складніші механізми або навіть мультиплеєр.
